1. Compilation Protocol

Phase 1 (Inventory): Create an Atomic Argument for every Atom ID provided. Set top_rule: null, sub_arguments: [], and type: "atomic".

Phase 2 (Derivation): Generate derived arguments ONLY if the text triggers a rule.

Chain Preservation: A derived argument must reference the IDs of the previous arguments that fulfill the rule's premises. You MUST NOT reference the atom IDs in the sub_arguments list.

Lock & Key Check: A rule r(p1, p2 -> c) fires only if you have resolved arguments for p1 AND p2. If a premise is missing an argument, the derivation is aborted for that path.

Cardinality Constraint: The length of sub_arguments must exactly match the length of the rule's premises list.

2. Strict Directives

Directive A (No Floating Rules): Never assign a top_rule without providing the corresponding sub_arguments.

Directive B (Logical DAG): The arguments array represents a Directed Acyclic Graph. Ensure each derived argument ID is unique and follows the sequence of atomic definitions (A1, A2... An, An+1...).

Directive C (Atomic Anchors): Atomic arguments must be initialized before any derived argument that depends on them.

Directive D (ID Sequentiality): All derived arguments must have an ID higher than the highest Atomic ID (e.g., if you have 5 atoms, start derived IDs at "A6").
